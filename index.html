Microsoft Fabric Capacity Estimator

A planning-grade calculator for sizing Fabric capacity at the CU level. It lets you model migration style (pipelines, notebooks, dataflows, mirroring, warehouse loads, shortcuts), storage growth, time horizon, semantic-model/report mix, Lakehouse vs Warehouse gold layer, DAX complexity, real-time analytics, and ML. The result is a recommended F SKU plus a detailed explanation of the workload components driving that recommendation.

This is a heuristic estimator, not an official Microsoft sizing engine. Use it for planning, then calibrate with the Fabric Capacity Metrics app after a pilot.

How the estimator works

The calculator projects future data volume, derives daily processed volume, allocates that work across Fabric engines, estimates ETL/ELT compute, adds semantic-model/reporting load, then applies safety and burst headroom before mapping the result to the nearest F SKU. CU estimates are intentionally conservative and editable through the planning assumptions controls.

Suggested next step: run your first migrated domain on a temporary Fabric capacity, compare this model with the Capacity Metrics app, then adjust the calibration multiplier and workload coefficients before buying a long-term reserved capacity.
